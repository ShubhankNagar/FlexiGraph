# Security Policy

## Supported Versions

| Version | Supported          |
| ------- | ------------------ |
| 1.x.x   | :white_check_mark: |

## Reporting a Vulnerability

We take security seriously. If you discover a security vulnerability, please follow these steps:

### Do NOT

- Open a public GitHub issue
- Post about it on social media
- Disclose it publicly before it's fixed

### Do

1. **Email us directly** at [shubhanknagar@gmail.com] with:
   - Description of the vulnerability
   - Steps to reproduce
   - Potential impact
   - Any suggested fixes (optional)

2. **Wait for our response** (usually within 48 hours)

3. **Work with us** to understand and resolve the issue

## What to Expect

- **Acknowledgment**: Within 48 hours
- **Initial Assessment**: Within 1 week
- **Resolution Timeline**: Depends on severity
  - Critical: 24-48 hours
  - High: 1 week
  - Medium: 2-4 weeks
  - Low: Next release

## Security Best Practices

When using ngx-flexigraph:

1. Always use the latest version
2. Keep dependencies updated
3. Sanitize any user input before passing to the library
4. Review exported data before sharing externally

## Acknowledgments

We appreciate security researchers who help keep our community safe. Contributors who report valid vulnerabilities will be acknowledged in our release notes (unless they prefer to remain anonymous).
